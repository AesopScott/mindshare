# Improve

## Purpose

Turn evaluation results and production observations into prioritized improvements to prompts, tools, memory, policies, tests, and workflow.

## Inputs

- Observation reports
- Eval failures
- User feedback
- Incidents and support notes

## Activities

- Cluster failures by root cause.
- Prioritize improvements by impact and risk.
- Update prompts, tools, evals, context, or design.
- Record what changed and why.

## Outputs

- Improvement backlog
- Updated eval cases
- Change log
- Next experiment plan

## Done Criteria

- Improvements are tied to evidence.
- Changes have regression coverage where practical.
- The next iteration has clear priorities.
- Lessons are fed back into earlier MAPS phases.
