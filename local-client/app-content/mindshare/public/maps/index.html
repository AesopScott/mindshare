Understanding MAPS.

MAPS is the Multi-Agent Pipeline Skills lifecycle — a phased path from M0 Foundation through M11 Improve for designing, building, and operating agent systems. This page explains the lifecycle and shows how to install any phase skill straight from the Tools page into Claude Code.

account_tree Open the MAPS Skills catalog
The Tools workflow

Install a MAPS skill in four steps

Inside the MindShare app, the Tools page now ships the full MAPS skills catalog. Installing copies a skill — and the templates it depends on — into your Claude Code skills folder.

1

Open Tools → MAPS Skills

In the app nav, open Tools and select MAPS Skills in the left sidebar. The catalog opens in the main panel.

2

Browse the lifecycle

Every phase M0–M11 shows its Base skill and any Augmented variant. M5 expands the nested agent pipeline A1–A8.

3

Install one or install all

Click Install on any skill, or Install all skills at the top. Each install copies the full skill folder plus its related templates to ~/.claude/skills/ and ~/.codex/skills/, overwriting any existing copy.

4

Use it in Claude Code

Installed skills are immediately available to Claude Code — invoke a phase like /foundation or /multi-agent-roster to run that step of the pipeline.

info Install runs through the MindShare desktop app. Opened in a plain browser, the catalog is read-only and the Install buttons are disabled.

What an install actually copies

folder_open

The whole skill folder

SKILL.md plus any bundled templates/ and references/ the skill ships with.

link

Related templates

Any shared MAPS template the skill references is pulled in too, so each installed skill is self-contained.

home_storage

Into both skills folders

Everything lands under ~/.claude/skills/<name>/ and ~/.codex/skills/<name>/, overwriting older copies so installs stay current.
