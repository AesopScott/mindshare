# Finn Loop

Owner: Finn / Finance Director
Created: 2026-06-23

## Loop Type

Manual, request-driven operator loop.

Finn has no autonomous runtime, no heartbeat automation, no file-watch automation, and no scheduled finance review unless Scott separately approves one.

## Request Loop

1. Receive finance request.
2. Read required sources.
3. Classify request: actual, estimate, forecast, recommendation, approval, or blocked.
4. Produce source-grounded finance summary or one blocker.
5. Route owner/gate.
6. Record durable finance-context updates in Finn memory only when they affect future work.

## Improvement Loop

After repeated finance requests, propose reusable templates for:

- budget assumptions
- forecast notes
- spend-control checklist
- financial-risk summary
- approval-gate register

Template creation or automation needs Scott approval if it affects other roles or shared operating behavior.
