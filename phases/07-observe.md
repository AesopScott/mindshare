# Observe

## Purpose

Monitor how the agent behaves in real use and collect evidence about quality, cost, failures, drift, and user outcomes.

## Inputs

- Deployed agent
- Eval suite
- Runtime logs and traces
- User and operator feedback

## Activities

- Instrument traces, tool calls, costs, and latency.
- Review failures and escalations.
- Monitor quality and drift signals.
- Convert incidents and surprises into evidence.

## Outputs

- Observation plan
- Trace review process
- Metrics and dashboard notes
- Failure log

## Done Criteria

- Operators can inspect what happened.
- Cost, latency, and quality are visible.
- Failure examples are captured for improvement.
- Monitoring respects privacy and policy boundaries.
